# FlexSim MCP 项目计划大纲

> 目标：通过模型上下文协议（MCP）控制和自动化 FlexSim 仿真，实现从自然语言描述到仿真模型的端到端自动化，无需手动搭建模型。
>
> FlexSim 文档：https://docs.flexsim.com/en/26.0/
> FlexSimPy SDK 版本：25.0.11

---

## 一、可行性评估

### ✅ 技术可行性（已验证）

| 层级 | 技术 | 状态 |
|------|------|------|
| MCP 协议层 | FastMCP (Python) | 成熟，已验证 |
| Python ↔ FlexSim 桥接 | FlexSimPy SDK 25.0.11 (.pyd) | 官方提供，支持 Python 3.9–3.12 |
| FlexSim 模型编程构建 | 通过 `controller.evaluate()` 执行 FlexScript | 核心桥梁，所有模型操作均通过此接口 |
| AI 驱动的意图理解 | Claude/GPT via MCP Client | 原生支持 |

### ⚠️ 已知约束

- **仅限 Windows**：FlexSimPy 是 Windows `.pyd` 扩展，无跨平台方案
- **需要 FlexSim 许可证**：本地安装必须有效 license（或 `evaluationLicense=True` 评估模式）
- **FlexSimPy 需编译**：依赖 MSBuild + VS Build Tools 2022，需与本地 Python 版本匹配（支持 3.9/3.10/3.11/3.12）
- **当前项目的核心目标**：通过 MCP 工具从自然语言描述生成新 FlexSim 模型

### 🔑 关键技术发现：FlexSimPy 的 `evaluate()` 通道

FlexSimPy SDK 仅暴露 12 个高层 Python 方法（见附录 A），所有复杂模型操作（创建对象、编辑属性、连接端口等）必须通过 `controller.evaluate("FlexScript代码")` 发送 FlexScript 字符串到引擎执行。这意味着：

1. **模型构建的本质**是动态生成 FlexScript 代码字符串
2. **MCP 工具**负责将高层语义（"添加一台处理器"）翻译为正确的 FlexScript 调用
3. **错误处理**必须捕获 `evaluate()` 返回值并解析

#### 2.2 通过 `evaluate()` 执行的 FlexScript 模型构建 API

```python
# 1. 创建 3D 对象（从库中实例化到模型）
controller.evaluate("""
    Object source = createinstance(library().find("Source"), model());
    source.name = "CustomerArrival";
    source.setLocation(0, 0, 0);
""")

# 2. 创建并定位处理器
controller.evaluate("""
    Object proc = createinstance(library().find("Processor"), model());
    proc.name = "Machine1";
    proc.setLocation(20, 0, 0);
""")

# 3. 端口连接（建立流向关系）
controller.evaluate("""
    Object src = model().find("CustomerArrival");
    Object proc = model().find("Machine1");
    src.outputConnect(proc);
""")

# 4. 设置对象属性（通过节点树路径或 label）
controller.evaluate("""
    Object proc = model().find("Machine1");
    proc.labels["processTime"].value = 5.0;
""")

# 5. 查询模型中所有对象
result = controller.evaluate("""
    string s = "";
    for (int i = 1; i <= model().subnodes.length; i++) {
        s += model().subnodes[i].name + "\\n";
    }
    return s;
""")
```

#### 2.3.1 builder/translator/model_spec 模块职责与接口

- `mcp_server/flexsim_builder.py`（建模核心封装）：
  - 面向 MCP 工具的**内部 API**，不直接暴露给外部客户端。
  - 关键函数示例（接口设计层面）：
    - `create_object(controller, *, type: str, name: str, x: float, y: float, z: float, params: dict | None = None) -> dict`

    - `validate_model(controller) -> list[str]`

- `mcp_server/model_spec.py`（精简版 ModelSpec 定义 + 校验）：
  - 提供 ModelSpec 的 Python 数据类/TypedDict 以及 JSON Schema（可选）。
  - 关键能力：
    - `parse_model_spec(obj: dict) -> ModelSpec`
    - `validate_model_spec(spec: ModelSpec) -> list[str]`（返回问题列表，空列表表示通过）。
- `mcp_server/translator.py`（ModelSpec → FlexScript 调用序列）：
  - 面向 builder 层，将结构化 ModelSpec 翻译为对子函数 `create_object` / `connect_objects` / `apply_params` 的一系列调用。
  - Phase 2 中可以仅支持**单一场景**（simple_production_line）的最小子集，保证接口稳定即可。

#### 2.6 错误处理与模型验证

```python
class ModelValidationError(Exception):
    """模型验证失败时抛出"""
    pass

def safe_evaluate(controller, script: str) -> str:
    """安全执行 FlexScript，捕获引擎错误"""
    try:
        result = controller.evaluate(script)
        if result is None:
            raise ModelValidationError(f"FlexScript 执行返回 None: {script[:100]}")
```

#### 2.7 Phase 2 精简版 ModelSpec 结构与校验规则

在 Phase 2 中仅使用一个**精简版** ModelSpec 作为过渡层，字段如下：

```json
{
  "model_name": "simple_production_line",
  "objects": [
    { "type": "Source", "name": "Source1", "x": 0,  "y": 0,  "z": 0, "params": { "interArrivalTime": "exponential(0, 2, 0)" } },
    { "type": "Queue",  "name": "Buffer",  "x": 10, "y": 0,  "z": 0 },
    { "type": "Processor", "name": "Machine1", "x": 20, "y": 0, "z": 0, "params": { "processTime": "exponential(0, 3, 0)" } },
    { "type": "Sink",   "name": "Exit",    "x": 30, "y": 0,  "z": 0 }
  ],
  "connections": [
    { "from": "Source1", "to": "Buffer" },
    { "from": "Buffer",  "to": "Machine1" },
    { "from": "Machine1","to": "Exit" }
  ],
  "simulation": {
    "runTime": 3600
  }
}
```

**校验规则（示意）：**

- 必填字段：
  - 根级：`objects` 非空列表；`connections` 可空但建议非空；`simulation.runTime` 必须为正数。
  - 对象级：`type`, `name`, `x`, `y`, `z` 必填。
- 类型白名单：
  - `type` 必须在固定资源白名单内（Phase 2 初期：`Source` / `Queue` / `Processor` / `Sink`）。
- 连接合法性：
  - `connections[].from` / `connections[].to` 必须引用 `objects[].name` 中已存在的名称。
  - 不允许自连接（`from == to`）。
- 拓扑基本检查（交由 `validate_model_spec` 与运行后 `validate_model` 双层保障）：
  - 至少存在一个 `Source` 与一个 `Sink`。
  - 所有固定资源要么有输出连接，要么被明确标记为终点（Sink）。

> 说明：Phase 2 中 ModelSpec 的职责主要是**约束与文档化**构建输入，完整自然语言到 ModelSpec 的映射留待 Phase 3 实现。

#### 2.8 Phase 2 最小可行构建流程与验收标准（simple_production_line）

结合前文工具与模块设计，Phase 2 的技术验收流程定义为：

1. MCP 客户端调用 `flexsim_launch`（如有需要）与 `flexsim_new_model`，确保在一个干净模型上操作。
2. 通过以下两种方式之一构建 `simple_production_line` 场景：
   - **方式 A：显式调用工具**：按顺序调用
     - `flexsim_add_object(type="Source", name="Source1", x=0, y=0, z=0, params={...})`
     - `flexsim_add_object(type="Queue", name="Buffer", x=10, y=0, z=0)`
     - `flexsim_add_object(type="Processor", name="Machine1", x=20, y=0, z=0, params={...})`
     - `flexsim_add_object(type="Sink", name="Exit", x=30, y=0, z=0)`
     - `flexsim_connect_objects(source="Source1", target="Buffer")`
     - `flexsim_connect_objects(source="Buffer", target="Machine1")`
     - `flexsim_connect_objects(source="Machine1", target="Exit")`
   - **方式 B：模板构建**：调用 `flexsim_build_from_template(template_name="simple_production_line", params={...})` 一次性完成。
3. 调用 `flexsim_validate_model`，预期返回问题列表为空或仅含提示级信息。
4. 调用 `flexsim_reset`，随后使用 `flexsim_run_to_time(time=3600)` 运行 1 小时仿真。
5. 使用 `flexsim_get_performance_measure(name="Output")`（或等价 KPI）获取吞吐量，并能通过 MCP 返回到上层客户端。

只要上述步骤可以在**无人工干预 GUI** 的情况下完整执行并返回合理的输出，即视为 Phase 2 成功达标，为 Phase 3 的自然语言建模提供可靠的模型构建基础。

#### 3.1 结构化中间表示层（ModelSpec）

自然语言不直接映射到 FlexScript。引入 JSON 中间表示层，AI 先生成结构化规格，再由代码翻译为 FlexScript 调用：

```json
{
  "model_name": "超市收银台仿真",
  "objects": [
    {"type": "Source", "name": "Customers", "x": 0, "y": 0, "params": {"interArrivalTime": "exponential(0, 2, 0)"}},
    {"type": "Queue",  "name": "WaitingLine", "x": 10, "y": 0},
    {"type": "Processor", "name": "Cashier1", "x": 20, "y": 5, "params": {"processTime": "exponential(0, 3, 0)"}},
    {"type": "Processor", "name": "Cashier2", "x": 20, "y": 0, "params": {"processTime": "exponential(0, 3, 0)"}},
    {"type": "Processor", "name": "Cashier3", "x": 20, "y": -5, "params": {"processTime": "exponential(0, 3, 0)"}},
    {"type": "Sink",   "name": "Exit", "x": 30, "y": 0}
  ],
  "connections": [
    {"from": "Customers", "to": "WaitingLine"},
    {"from": "WaitingLine", "to": "Cashier1"},
    {"from": "WaitingLine", "to": "Cashier2"},
    {"from": "WaitingLine", "to": "Cashier3"},
    {"from": "Cashier1", "to": "Exit"},
    {"from": "Cashier2", "to": "Exit"},
    {"from": "Cashier3", "to": "Exit"}
  ],
  "simulation": {"runTime": 3600, "replications": 1}
}
```

**好处**：
- JSON 可被 **schema 校验**，在发送到 FlexSim 之前就发现问题（缺少 Sink、孤立对象等）
- AI 生成 JSON 比生成 FlexScript 字符串**更可靠、更易调试**
- 可序列化保存、版本对比、参数扫描时批量修改

#### 3.2 ModelSpec → FlexScript 翻译器

```python
class ModelSpecTranslator:
    """将 ModelSpec JSON 翻译为一系列 FlexScript evaluate() 调用"""
    
    def translate(self, spec: dict, controller) -> None:
        # 1. 校验 spec（JSON Schema）
        self.validate(spec)
        # 2. 按顺序创建对象
        for obj in spec["objects"]:
            self.create_object(controller, obj)
        # 3. 建立连接
        for conn in spec["connections"]:
            self.connect_objects(controller, conn)
        # 4. 应用参数
        for obj in spec["objects"]:
            if "params" in obj:
                self.apply_params(controller, obj)
        # 5. 验证模型
        issues = validate_model(controller)
        if issues:
            raise ModelValidationError(f"模型验证失败: {issues}")
```

## 七、并发与会话管理

FlexSim 引擎是单实例单线程运行，多个 MCP 客户端同时调用会导致冲突。

```python
import threading

class FlexSimSessionManager:
    """确保同时只有一个操作访问 FlexSim 引擎"""
    
    def __init__(self):
        self._lock = threading.Lock()
        self._controller = None
    
    def execute(self, operation):
        """串行化所有对 FlexSim 的操作"""
        with self._lock:
            return operation(self._controller)
```

**设计原则**：
- 所有 MCP 工具函数通过 `SessionManager.execute()` 访问 controller
- 模型构建操作作为**原子事务**：若中途失败则回滚（重新打开干净模型）
- 长时间仿真运行时返回状态轮询接口，避免 MCP 调用超时
