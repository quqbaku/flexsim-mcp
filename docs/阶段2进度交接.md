## 3. 当前已知问题（最关键阻塞点）

### 3.1 FlexSimPy `evaluate()` 通道能力不一致（阻塞 Phase 2）

**现象：**

- 节点路径求值可用：
  - 例如 `flexsim_evaluate("MAIN:/project/exec/time")` 返回数值（如 `0.0`）。
- FlexScript 脚本执行不可用：
  - `flexsim_evaluate("return time();")` 返回 `None`（已封装为 `""`，但本质仍失败）
  - `createinstance(library().find("Source"), model())` 返回 `None`，模板构建失败
  - 任何依赖 `model().subnodes` 的树遍历脚本也返回 `None`

**根本原因（推测）：**

FlexSimPy 的 `evaluate()` 在无 GUI 模式下（`showGUI=False`）可能仅支持节点路径表达式，不支持完整 FlexScript 语句。这是 SDK 设计限制，而非代码 bug。

**临时缓解（已做）：**

- 对 Phase 1 工具：`FlexSimClient.evaluate()` 对 `None` 不再直接抛错，返回空字符串，避免旧工具中断。
- 对 Phase 2 builder：仍要求关键建模脚本返回值（`allow_none=False`），避免误判成功。

### 3.2 `send()/receive()` 通道挂起问题

官方 `TestFlexSimPy.py` 使用 `send/receive` + `TestSendReceive.fsm` 是唯一已知可用的双向通信方案。但：

- 当前直接调用 `controller.send()` / `controller.receive()` 会**挂起或段错误**，原因是没有加载具备 `OnMessage` 触发器的模型。
- 解决路径：先手动创建 `executor.fsm`（含 `OnMessage` 循环），再开放 `execute_script()` 通道。

### 3.3 US-001/US-002 循环依赖

- 创建 `executor.fsm` 需要 `evaluate()` 执行 FlexScript（但 evaluate 本身不可用）
- 测试 `execute_script()` 需要 `executor.fsm` 已存在
- **打破循环的唯一方法**：用 FlexSim GUI 手动创建并保存 `executor.fsm`（参考 `docs/executor-fsm-guide.md`）

## 5. 最小复现与定位步骤（交接用）

建议接手者先跑以下探针，快速判断当前 FlexSim 环境：

1. **验证官方示例**（最快排查方式）：
   ```
   cd FlexSimPy-25.0.11/TestFlexSimPy
   python TestFlexSimPy.py
   ```
   - 若官方 `TestSendReceive.fsm` 测试通过 → FlexSim 环境正常，问题在 `config.toml` 或路径配置
   - 若挂起/报错 → FlexSim 安装或许可证有问题

2. **验证节点路径通道**（应有数值返回）：
   - 调用：`flexsim_evaluate("MAIN:/project/exec/time")`

3. **验证 FlexScript 通道**（若返回空/None 则 Phase 2 阻塞）：
   - 调用：`flexsim_evaluate("return time();")`

4. **验证模板构建**（预期失败，除非 FlexScript 通道已修复）：
   - 调用：`flexsim_build_from_template(template_name="simple_production_line")`

若第 3 步失败，**立即转向手动创建 `executor.fsm` 方案**（参考 `docs/executor-fsm-guide.md`）。
