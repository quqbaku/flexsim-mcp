## 2026-04-10 - executor.fsm Validation Result (New Finding)
- **Attempted:** validate manually placed `executor.fsm` by running `python scripts/smoke_eval.py`
- **Observed:**
  - root-level `executor.fsm` is detected correctly
  - `execute_script('return time();')` times out
  - `execute_script("createinstance(...); return 'ok';")` also times out
- **Technical conclusion:**
  - the current `executor.fsm` design is still not compatible with FlexSimPy's message return mechanism
  - from `FlexSimPy.cpp`, `controller.receive()` only unblocks when FlexSim-side code pushes into the controller receive queue via `sendToController(...)`
  - a plain `OnMessage` trigger with `sendtomain(result)` is therefore not sufficient to satisfy the current Python-side `receive()` contract
- **Implication:**
  - US-001 is not solved merely by placing an `executor.fsm` file in the correct directory
  - the remaining blocker is now specifically the **message bridge design inside executor.fsm**, not file placement or launch configuration
- **Status:** Blocker refined again - executor file exists, but protocol is still mismatched

## 关键阻塞与建议

### 根本阻塞原因
FlexSimPy 在当前环境下：
- `controller.evaluate(script)` 对节点路径可用，但对 FlexScript 建模脚本仍只返回空结果，无法完成对象创建和模型保存的验收链路
- `controller.send()` / `controller.receive()` 的调用时序问题已修复，但缺少 executor.fsm 仍无法完成真正验收

  - 参考 `docs/executor-fsm-guide.md`，在 FlexSim GUI 中添加 `OnMessage` 触发器并保存到项目根目录

2. **P1 - 立即验证 US-002**
  - 一旦 executor.fsm 存在，直接运行 `python scripts/smoke_eval.py`
  - 重点看 `execute_script('return time();')` 和 `createinstance(...); return 'ok';` 两项

3. **P2 - 验证 US-004 / US-005**
  - 运行 `python scripts/smoke_eval.py` → 验证 US-005 集成侧
  - 运行 `flexsim_build_from_template('simple_production_line')` → 验证 US-004

4. **P3 - 补充 official sample 对照验证**
